# User Stories

## US01 – Acessar Atividade

**Como** aluno,
**desejo** acessar uma atividade,
**para** visualizar suas instruções.

### Critérios de Aceitação

* O aluno deve estar matriculado na turma.
* A atividade deve estar publicada.
* O sistema deve exibir descrição, prazo e anexos.

---

## US02 – Realizar Atividade

**Como** aluno,
**desejo** realizar uma atividade,
**para** concluir minhas tarefas acadêmicas.

### Critérios de Aceitação

* O aluno deve conseguir visualizar os requisitos da atividade.
* O aluno deve poder preparar sua resposta antes do envio.

---

## US03 – Enviar Atividade

**Como** aluno,
**desejo** enviar minha atividade,
**para** receber avaliação do professor.

### Critérios de Aceitação

* O sistema deve permitir anexar arquivos.
* O sistema deve registrar data e hora da entrega.
* O sistema deve confirmar o envio realizado.

---

## US04 – Visualizar Envios

**Como** professor,
**desejo** visualizar os envios realizados pelos alunos,
**para** iniciar o processo de correção.

### Critérios de Aceitação

* O professor deve visualizar a lista de entregas.
* O sistema deve informar a data de envio.
* O sistema deve indicar entregas realizadas após o prazo.

---

## US05 – Corrigir Atividades

**Como** professor,
**desejo** corrigir atividades,
**para** avaliar o desempenho dos estudantes.

### Critérios de Aceitação

* O professor deve acessar os arquivos enviados.
* O professor deve registrar a avaliação da atividade.
* O sistema deve armazenar o resultado da correção.

---

## US06 – Atribuir Nota

**Como** professor,
**desejo** atribuir notas,
**para** registrar o resultado da avaliação.

### Critérios de Aceitação

* A nota deve estar dentro do intervalo definido pela instituição.
* O sistema deve salvar a nota atribuída.
* O aluno deve poder consultar a nota posteriormente.

---

## US07 – Registrar Feedback

**Como** professor,
**desejo** registrar feedback,
**para** orientar os alunos sobre seu desempenho.

### Critérios de Aceitação

* O professor deve poder adicionar comentários.
* O feedback deve ficar vinculado à atividade corrigida.
* O aluno deve conseguir visualizar o feedback.

---

## US08 – Receber Notificação de Entrega

**Como** professor,
**desejo** ser notificado quando uma atividade for enviada,
**para** acompanhar novas entregas.

### Critérios de Aceitação

* O sistema deve gerar uma notificação após cada envio.
* A notificação deve identificar o aluno e a atividade.
* O professor deve conseguir visualizar a notificação.

---

## US09 – Receber Notificação de Nota

**Como** aluno,
**desejo** ser notificado quando minha nota estiver disponível,
**para** acompanhar meu desempenho.

### Critérios de Aceitação

* O sistema deve enviar uma notificação após a publicação da nota.
* A notificação deve indicar qual atividade foi corrigida.
* O aluno deve conseguir acessar a atividade diretamente pela notificação.

---

## US10 – Visualizar Nota e Feedback

**Como** aluno,
**desejo** visualizar minha nota e feedback após a correção,
**para** compreender meu desempenho e identificar pontos de melhoria.

### Critérios de Aceitação

* O sistema deve exibir a nota recebida.
* O sistema deve exibir o feedback do professor.
* Apenas o próprio aluno deve visualizar seus resultados.
